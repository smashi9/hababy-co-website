## Selected UI Direction

The selected direction is:

```text
Direction 2 — The Warm Family Service
```

## Why This Direction Was Selected

Direction 2 is the strongest fit for Version 1 because Hababy & Co is a request-first, parent-led baby gear rental service.

The website should not feel like an automated marketplace or instant checkout platform. It should feel like a warm, careful, human service that personally checks availability, safety, cleanliness, and delivery details before confirming a request.

This direction turns the manual confirmation process into a strength.

Instead of feeling like:

```text
The system is not automated yet.
```

it should feel like:

```text
A real person is carefully checking everything for my family.
```

## Desired User Impression

A visitor should feel:

```text
This is run by real parents or a caring local team who understand what it is like to travel with a baby. The gear will be clean, safe, and personally prepared. I can request what I need, and someone will confirm the details with me.
```

The emotional tone should be:

* Warm
* Calm
* Trustworthy
* Practical
* Parent-led
* Premium but approachable
* Mobile-first
* Human, not corporate
* Rabat-specific without feeling touristy

## Core UI Principle

The design must make the request-first model feel intentional.

Use language like:

```text
Request a booking
We confirm availability
Personally prepared
Delivered before you arrive
```

Avoid language like:

```text
Book now
Buy now
Instant checkout
Pay securely online
Guaranteed availability
```

## 1. Header Navigation

## Purpose

The header should make the brand feel calm, trustworthy, and easy to contact.

## Layout

Desktop:

* Warm Off-White background
* Left-aligned horizontal logo
* Navigation links centered or right-aligned
* Language switch visible
* WhatsApp contact link visible

Mobile:

* Logo left
* Menu or simplified nav right
* WhatsApp CTA accessible
* No clutter

```text
/brand/hababy-logo-horizontal.svg
```

```text
Rent Gear

Welcome Kits
```

## 2. Hero Section

## Purpose

The hero should immediately explain what Hababy & Co does and reassure the visitor that this is a human-confirmed service.

## Visual Direction

Use a warm Sand background band.

The hero should feel friendly, clear, and premium.

## Suggested Headline

```text
Baby gear rental in Rabat, delivered before you arrive.
```

## Suggested Subheadline

```text
Travel lighter with clean, carefully prepared baby and toddler essentials — personally confirmed before every delivery.
```

## Optional Script Accent

Use Quicker Script very sparingly.

Example:

```text
Personally prepared for your little ones
```

This should be small and decorative, not a main headline.

## Hero CTAs

Primary:

```text
Request a booking
```

Secondary:

```text
Chat on WhatsApp
```

Do not use:

```text
Book now
Pay now
Checkout
```

## Hero Visual

Use a warm, authentic lifestyle image or a clean product/arrival visual.

If no real images are available yet, use a calm placeholder area with brand styling rather than stock imagery that feels fake.

## 3. How It Works

## Purpose

This section should appear early because the service is request-first.

It should reduce anxiety by explaining the process clearly.

## Recommended Steps

```text
1. Choose what you need
2. Send a request
3. We confirm availability within 24 hours
4. We arrange payment, delivery, and pickup
```

## Important Copy Rule

Make it clear that:

```text
Submitting a request does not instantly confirm the booking.
```

Suggested wording:

```text
Every request is personally checked so we can confirm the right items, timing, and delivery details for your family.
```


If font setup is not yet implemented, use safe fallbacks first and add proper font loading during implementation.

Do not overuse the script font.

## Component Rules

## Cards

Default cards should use:

```text
Background: #FFFFFF
Border: 1px solid #A89E94
Radius: rounded-2xl / approximately 16px
Padding: generous, mobile-friendly
Shadow: none or very soft low-opacity only
```

Cards should feel warm, clean, and touch-friendly.

## Buttons

Primary button:

```text
Marrakech Red background
White text
Rounded shape
Label: Request a booking
```

Secondary button:

```text
Transparent or Warm Off-White background
Taupe border
Charcoal text
Label: Chat on WhatsApp
```

Do not create payment CTAs.

## Badges

Use badges for service states, such as:

```text
Cleaned & inspected
Personally confirmed
Rabat pilot
Model image
```

Use Sage sparingly for organic/premium consumable badges.

## Product Preview Rules

Product previews should show:

* Product name
* Slug or category if useful
* Starting price in MAD
* Availability mode
* Deposit if included
* Clear request CTA later

Product previews should not imply guaranteed instant availability.

Car seats should feel more carefully handled and may show:

```text
Personally confirmed
```

## Copy Rules

Tone should be:

* Warm
* Helpful
* Clear
* Parent-led
* Reassuring
* Practical

Avoid:

* Overly cute language
* Marketplace language
* Luxury clichés
* Instant checkout language
* Payment gateway language
* Overpromising availability

Use:

```text
Request a booking
We confirm availability
Prepared before delivery
Payment and deposit arranged before handover
```

Do not use:

```text
Book now
Buy now
Pay now
Instant checkout
Guaranteed today
```

## Mobile Rules

The site must be mobile-first.

On mobile:

* CTAs must be easy to tap
* Long grids can become horizontal scroll sections
* Header should stay simple
* Text should not feel cramped
* Product cards should be readable
* WhatsApp access should not block important content
* The request-first process should be visible early

## Acceptance Criteria

This UI handoff is successfully implemented when:

* The default Next.js starter homepage is removed.
* The homepage feels like Hababy & Co, not a generic template.
* The horizontal logo appears in the header.
* The page uses the warm brand palette.
* The homepage explains the request-first model clearly.
* The hero has “Request a booking,” not “Book now.”
* The How It Works section appears early.
* The Trust & Safety section is visible and reassuring.
* Product preview data can display from Supabase if used.
* The layout works on mobile.
* No online payment language or UI appears.
* Lint and build pass.
* Vercel deploys successfully.

## Open UI Questions

These do not block implementation, but should be resolved before launch:

1. What real photography will be used?
2. Should the homepage hero use lifestyle photography or a product/arrival illustration?
3. Should the WhatsApp link say “Chat on WhatsApp” or “Ask a Parent”?
4. Should bundles appear before or after individual products on the homepage?
5. Should the homepage mention airport delivery immediately or only inside delivery zones?
6. When should French content be added?
7. Should horizontal carousels be implemented now or simple responsive grids first?
8. Should the stork mark be used as a trust seal or reserved for favicon/badges?
9. How much of the placeholder seed copy should be replaced before first public sharing?
